# Engineering Onboarding Checklist

## Day 0
- Access repository and tooling.
- Run `./scripts/bootstrap.sh` successfully.
- Run `./scripts/check.sh` successfully.
- Read `README.md` and `CONTRIBUTING.md`.

## Week 1
- Ship one small scoped change.
- Run `./scripts/check.sh` locally before PR.
- Participate in one PR review.

## Week 2+
- Pick a LEP-tagged issue and own delivery end-to-end.
- Document one process improvement in repository docs.
