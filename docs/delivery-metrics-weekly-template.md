# Weekly Throughput and Lead-Time Report Template

## Purpose
Track execution flow for active engineering work and surface trend shifts early.

## Cadence and Ownership
- Cadence: Weekly, published every Friday by 17:00 UTC.
- Owner: Founding Engineer (Software Engineer).
- Escalation reviewer: CTO.

## Reporting Window
- Week ending: `YYYY-MM-DD`
- Window start: `YYYY-MM-DD`
- Window end: `YYYY-MM-DD`

## Inputs
- Source issue board: Paperclip engineering issues for LEP.
- Included statuses for throughput: issues moved to `done` during window.
- Lead-time definition: elapsed time from `in_progress` entry to `done`.

- 4-week rolling average throughput: `N`

## Lead Time

  - `none` or list issue identifiers and owners
- Main variance drivers:
  - concise bullet(s)
- Planned adjustment for next week:
  - concise bullet(s)
